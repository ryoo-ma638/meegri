# meegri 〜ミーグリ練習シミュレーター〜

アイドルの「ミーグリ（オンライン・ミート＆グリート）」を、本番そっくりに練習できるWebアプリ。
持ち時間・通信ラグ・時間切れ・緊張、そしてアイドルの機嫌や反応まで再現します。

- フロントは **単一HTML（`app/index.html`）**。ビルド不要。
- アイドルは **インラインSVGの立ち絵**（IP安全な架空キャラ）、背景は **WebGL/GLSLシェーダー**。
- 会話は **2段構え**：① 標準は内蔵の「台本＋ルール」エンジン ② `.env` にキーを入れると **Gemini による本物のAI会話**（失敗時は自動で台本にフォールバック）。
- カメラ機能：本番で自分の顔を **WebGLビューティー** 付きで表示（**端末内のみ・アップロードなし**）。

## 遊び方（3通り）

### A. いちばん簡単（AIなし・台本モード）
`app/index.html` を**ダブルクリック**するだけ。
※カメラとAI会話を使うには下記 B / C（セキュアな文脈が必要）。

### B. ローカルでAI会話つき（おすすめ）
1. Gemini APIキーを無料取得 → https://aistudio.google.com/apikey
2. キーを設定（**自分だけが扱う**。誰にも渡さない）:
   ```sh
   cp .env.example .env
   #  .env を開いて GEMINI_API_KEY=取得したキー を記入
   ```
3. 起動（Node 18+。インストール不要）:
   ```sh
   node server.js
   ```
4. ブラウザで **http://localhost:8753** を開く（カメラ許可でビューティーも有効）。

### C. ネットに公開（GitHub + Vercel）
- リポジトリを Vercel にインポート → **Settings › Environment Variables** に `GEMINI_API_KEY` を登録（**コードには書かない**）。
- `api/reply.js` がサーバーレス関数として動き、AI会話が有効に。
- ※静的のみ（GitHub Pages 等）でも台本モードでフル프レイ可能。

## 🔑 APIキーの扱い（重要）
- キーは **サーバー側（`.env` / Vercelの環境変数）だけ** に置きます。**フロントのHTML/JSには一切書きません**。
- `.env` は `.gitignore` 済み → **GitHubに上がりません**。`.env.example` だけを共有用に置いています。
- 万一キーを公開してしまったら、すぐ Google AI Studio で**キーを再発行（無効化）**してください。

## 構成
```
app/index.html   フロント（ゲーム本体・単一HTML）
server.js        ローカル用サーバー（静的配信 + /api/reply・依存ゼロ）
api/reply.js     公開(Vercel)用のサーバーレス関数
.env.example     キー設定のテンプレ（.env は各自作成・非公開）
```

※登場するアイドルは架空のオリジナルキャラクターです。実在の人物・団体とは関係ありません。
